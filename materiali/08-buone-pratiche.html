Regole d'oro per lavorare con un agente AI

La regola di base

Più un'azione è difficile da annullare, più serve una conferma esplicita.

Claude Code per default chiede conferma su tutto ciò che non è pura lettura — l'hai visto più volte nel Video 1. È una scelta di sicurezza, non un fastidio da disattivare alla leggera.

Innocuo e reversibile

Leggere un file, controllare lo stato di git.

Non lo è

Cancellare un branch, forzare una modifica, pubblicare online.

Permessi: cosa automatizzare

Automatizza il rischio basso, chiedi sempre conferma sul resto.

Con /permissions, visto nel Video 3, decidi tu dove tracciare il confine.

Sì, automatizza
  • leggere file
  • eseguire test
  • controllare lo stato di git
No, chiedi sempre
  • push su branch principali
  • merge di pull request
  • cancellazione di file o branch
  • pubblicazione di build

/loop e /goal: attenzione in più

Usali solo dove va bene che Claude proceda da solo a lungo.

Se il compito include un'azione irreversibile, meglio spezzarlo: lascia che l'agente prepari tutto in autonomia, ma chiedigli di fermarsi prima del passaggio finale che non si può disfare.
