Video 4 - Skill e Plugin

Estendi Claude Code senza scrivere una riga di codice.

Una skill insegna a Claude una procedura. Un plugin la distribuisce, insieme ad altro, a tutto il team. In questo video vediamo entrambe in azione, a partire da Ponytail, già installata nel Video 1.

Cos'è una skill

Una procedura scritta che Claude legge e segue — non nuovo codice.

Ogni skill vive in un file SKILL.md: un frontmatter che descrive nome e scopo, seguito dalle istruzioni vere e proprie.

.claude/
skills/
nome-skill/
SKILL.md
--- name: nome-skill description: quando e come usarla --- Istruzioni che Claude legge e segue quando la skill viene invocata.

Dove vive una skill

Personale in ogni tuo progetto, oppure condivisa con la squadra.

Personale

Vive nella tua cartella utente. È disponibile in tutti i progetti su cui lavori, indipendentemente dal repository aperto.

Di progetto

Vive dentro il repository. Viaggia con git: chiunque clona il progetto ha automaticamente la stessa skill a disposizione.

Come si invoca una skill

Esplicita con /nome-skill, oppure riconosciuta da sola da Claude.

/nome-skill

La scrivi tu esplicitamente nel prompt: Claude la segue subito, dall'inizio alla fine.

Riconoscimento automatico

La tua richiesta corrisponde a quello che la skill descrive di fare: Claude la applica da solo, senza che tu la nomini.

> /plugin list
ponytail@ponytail — attiva (installata nel Video 1)

Ponytail in azione

Costringe Claude a scegliere la soluzione più semplice possibile.

Prima di scrivere, si chiede: serve davvero? esiste già qualcosa di più corto? sto aggiungendo flessibilità che nessuno ha chiesto?

lite full ultra
> aggiungi una funzione per calcolare lo sconto

Senza Ponytail

  • funzione generica per ogni caso futuro
  • parametri opzionali mai usati
  • una dipendenza in più per un calcolo semplice

Con Ponytail

  • una funzione, un parametro
  • nessuna dipendenza aggiunta
  • diff minimo, facile da rivedere

Cos'è un plugin

Un pacchetto più grande: skill, agenti, automazioni e MCP insieme.

Si distribuisce tramite un marketplace — un repository che elenca plugin installabili — e ogni comando è prefissato col nome del plugin per evitare conflitti.

Plugin
Skillprocedure scritte
Agentruoli dedicati
Hookautomazioni
MCP serverObsidian, Graphiti…

Installare un secondo plugin

Stesso meccanismo già visto per Ponytail nel Video 1.

Prima si aggiunge il marketplace, poi si installa il plugin specifico. Un procedimento identico ogni volta, qualunque sia il plugin.

> /plugin marketplace add anthropics/claude-plugins-official
Marketplace aggiunto
> /plugin install skill-creator@claude-plugins-official
Plugin installato
> /plugin list
ponytail@ponytail, skill-creator@claude-plugins-official

Gestire i plugin installati

/plugin list per vedere cosa hai, /plugin disable per metterlo in pausa.

/plugin listelenca tutti i plugin installati e il loro stato
/plugin disabledisattiva temporaneamente, senza disinstallare
