La gh CLI

Claude legge e scrive pull request senza che tu passi dal browser.

Dentro una sessione, l'agente ha accesso diretto alla gh, la command line ufficiale di GitHub: legge una pull request, i suoi commenti, il suo diff, e può crearne una nuova.

> mostrami i commenti sulla PR #12
Claude usa gh sotto il cofano per leggere i commenti
> gh pr create --title "..." --body "..."
PR creata — titolo e descrizione generati da Claude

La skill /review

Un secondo parere prima del merge, o prima della revisione umana.

Analizza le modifiche di una pull request GitHub cercando bug e problemi, usando lo stesso motore della revisione del codice locale.

> /review 12
Analizza il diff della PR #12, segnala bug e problemi trovati

@claude nei commenti

Una volta installata la GitHub App, basta un commento.

Con il comando /install-github-app, che ti guida passo passo nella configurazione, colleghi Claude al repository — una volta sola.

> /install-github-app
Ti guida passo passo nella configurazione della GitHub App

@claude in azione

Un commento su una issue, e Claude propone o applica la modifica.

@claude correggi il typo in questa riga
Claude legge il contesto, propone la modifica e crea un commit — tutto senza aprire un terminale

Perché è utile per tutto il team

Chi non sa git può proporre una correzione, senza doverlo imparare.

Resta comunque un passaggio umano di revisione prima che qualsiasi modifica venga unita al progetto — lo vediamo meglio nell'ultimo video.

Solo browser

Un commento con @claude su una issue o una PR: basta un account GitHub.

Terminale

gh CLI e /review, per chi lavora sul codice ogni giorno.
